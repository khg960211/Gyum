알고리즘 공부할 때 중요한 것
1. 시간
2. 메모리 공간

시간 복잡도 (Time complexity)
- 데이터가 많아질수록 걸리는 시간이 얼마나 급격히 증가하는가
- 시간 복잡도가 작을 수록 더 빠른 알고리즘이라고 한다.
- 시간복잡도와 관련된 수학 개념 : 로그, 거듭제곱, 1부터 n까지의 합

점근 표기법(Big-O Notation)
- 소요시간이 20n + 40 이라고 했을 때, n의 영향력이 더 큰것은 20n이다. 그러면 그냥 40을 없애고 앞의 숫자도 없애면
이 알고리즘은 O(n) -> 빅오 오브 앤 알고리즘 이라고 부른다.
- 2n제곱 + 8n + 157 -> n제곱 -> O(n제곱)   빅 오 오브 앤제곱 알고리즘
- O(n세제곱)
- O(lg n) 빅 오 오브 로그앤 알고리즘
- 점근 표기법의 핵심은 n이 엄청 크다고 가정하는 것이다. n이 별로 크지 않으면, 안좋은 알고리즘을 써도 문제가 없다.
n이 엄청 큰 경우에 알고리즘의 중요성을 실감하게 된다.
